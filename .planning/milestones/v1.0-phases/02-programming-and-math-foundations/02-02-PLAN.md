**lesson.md** — Written lesson covering:
- Part 1: Matrices as Transformations (15-20 min) — Matrices aren't just grids of numbers; they DO something to vectors. Analogy: a matrix is like a machine — feed it a vector, get a transformed vector out. Show what rotation, scaling, shearing look like. Dot product as "how similar are these two directions?" Matrix multiply as chaining transformations. Notation bridge: visual (quiver arrows) -> code (A * v) -> notation (matrix-vector multiplication).
- Part 2: Statistics — Making Sense of Data (15-20 min) — Mean as "the typical value," std as "how spread out," variance as std^2. Correlation as "do these move together?" Always with plots: histogram shape, mean line, std bands, scatter with r-value. Use relatable data (daily temperatures, ice cream sales, reaction times).
- Part 3: Eigenvalues — Directions That Survive (15-20 min) — Start with the transformation visualization from Part 1. Ask: "Most vectors change direction. But are there special vectors that KEEP their direction?" Those are eigenvectors. Lambda (eigenvalue) is how much they stretch. Notation bridge: visual (highlighted arrows) -> code ([V,D] = eig(A)) -> notation (Av = lambda*v). Keep it visual and intuitive — save covariance/PCA connection for Phase 4.

Follow established patterns: relatable analogies, visual-first, "Try it" prompts, chunked segments.

IMPORTANT pitfalls to cover (from research):
- * vs .* (matrix multiply vs element-wise multiply) — reinforce from Phase 1
- corrcoef returns a matrix, not a number — show R = corrcoef(x,y); r = R(1,2)
- Eigenvector sign/scaling ambiguity — "direction matters, not exact numbers"
- Eigenvalues can be complex for some matrices — we use symmetric/simple matrices here
