**lesson.md** -- Written lesson in 4 parts:
- Part 1: What Are Attractors? -- Ball-in-bowl metaphor: valleys are stable states, hills are unstable. A ball released on the landscape rolls to the nearest valley. In neural networks, valleys = stored memories, the "ball" = network state, "rolling" = state update rule. Analogy: "like a marble rolling to the bottom of a bowl -- the bowl shape is determined by what the network learned." Visual-first, equations AFTER.
- Part 2: Hopfield Networks -- Networks that store memories as patterns of +1/-1 activity. Hebbian learning: "neurons that fire together wire together" becomes W = (1/N) * patterns * patterns'. Retrieval: start from noisy pattern, update with S = sign(W*S), converge to nearest stored memory. Energy function E = -0.5 * S' * W * S always decreases. Capacity limit: ~0.14*N patterns.
- Part 3: Ring Attractors -- Neurons arranged in a circle, each with a preferred direction (connect to Phase 3 population coding!). Mexican-hat connectivity: neighbors excite, distant neurons inhibit. Result: a "bump" of activity that represents current head direction. Bump persists without input (memory), external input can shift it (updating).
- Part 4: Attractors in Neuroscience -- Brief connection to PCA: "Remember how PCA showed neural data lives in low dimensions? Attractor networks explain WHY -- the dynamics constrain activity to low-dimensional manifolds (valleys in the landscape)." Head direction cells, place cells, decision-making as attractor dynamics.

**demo_hopfield.m** -- Hopfield network implementation:
- rng(42), N = 25 (5x5 grid)
- Create 3 simple patterns: horizontal stripe (row 3 = +1, rest = -1), vertical stripe (column 3 = +1), diagonal (i,i positions = +1)
- Hebbian learning: W = (1/N) * (patterns * patterns'); W(1:N+1:end) = 0 (no self-connections)
- Retrieve from noisy version of pattern 1 (flip 30% of bits)
- Iterative update: S = sign(W * S); S(S==0) = 1 (break ties); stop when converged
- Track energy E = -0.5 * S' * W * S each iteration
- Figure 1: subplot(1,3,1) original pattern, subplot(1,3,2) noisy input, subplot(1,3,3) retrieved pattern. Use imagesc(reshape(..., 5, 5)) with gray colormap.
- Figure 2: Energy over iterations (should decrease monotonically)
- Figure 3: Overlap with all 3 stored patterns over iterations (show convergence to pattern 1)
- Print: overlap values, number of iterations to converge

IMPORTANT pitfalls:
- sign(0) = 0 in Matlab -- replace with +1 after sign()
- No self-connections: W(1:N+1:end) = 0
- Keep to 3 patterns for 25 neurons (capacity ~3.5)

**demo_ring_attractor.m** -- Ring attractor simulation:
- rng(42), N = 100 neurons on a ring
- theta = linspace(0, 2*pi, N+1); theta = theta(1:end-1) for neuron positions
- Mexican-hat connectivity: W(i,j) = cos(theta(i) - theta(j)) - 0.5; no self-connections
- Parameters: tau=1, dt=0.1, T=50 (500 steps)
- Initialize bump at pi/2: activity = exp(5*cos(theta - pi/2)); normalize
- Simulate: input = W * activity'; rectify with max(0,...); update activity; normalize each step
- Store activity_history for visualization
- Figure 1: imagesc heatmap (time x neuron preferred direction) showing stable bump
- Figure 2: subplot(1,2,1) initial bump profile, subplot(1,2,2) final bump profile (should be same position)
- Print: "Bump maintains position at ~90 degrees without external input"
- Brief mention: external input could shift the bump (head turning)
