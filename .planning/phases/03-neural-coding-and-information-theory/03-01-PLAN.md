**lesson.md** -- Written lesson covering:
1. Spike trains: "Neurons communicate with electrical pulses called spikes. A spike train is the sequence of when a neuron fires." Analogy: "Like a telegraph -- the message is in the timing." Explain Poisson firing (random with an average rate). Notation bridge: firing rate lambda, probability P(spike) = lambda * dt.
2. Tuning curves: "Different neurons respond to different things. A tuning curve shows how much a neuron fires for each stimulus." Analogy: "Like asking which radio station each antenna picks up best." Cosine tuning (Georgopoulos 1982). Notation bridge: f(theta) = r_0 + g * cos(theta - theta_pref).
3. Population coding: "One neuron is noisy and ambiguous. A group of neurons together gives a clear signal." Analogy: "Like asking 8 friends to point toward a sound -- average their pointing and you find it." Heatmap visualization.
4. Neural decoding: "Reading the neural code backwards -- from firing rates to stimulus." Population vector method. Notation bridge: weighted sum formula.

Follow established pattern: visual-first, small chunks, notation bridges after visual intuition.

**demo_spike_trains.m** -- Script that:
- Uses rng(42) for reproducibility
- Generates a Poisson spike train at 30 Hz for 2 seconds using rand(1, n_steps) < rate * dt
- Plots a raster plot for a single trial (stem or line-based)
- Generates 20 trials and plots a multi-trial raster plot
- Computes and plots a PSTH (peri-stimulus time histogram) using histcounts
- Computes and plots an interspike interval histogram
- Uses clear/clc/close all at top, descriptive comments, fprintf for key values

**demo_tuning_curves.m** -- Script that:
- Uses rng(42) for reproducibility
- Creates a neuron with preferred direction 90 degrees, baseline 10 Hz, gain 15 Hz
- Computes firing rates using cosd() for all directions 0:10:350
- Plots tuning curve with polarplot() showing the preferred direction clearly
- Also plots as a regular line plot (directions on x-axis, firing rate on y-axis) for clarity
- Creates 4 neurons with different preferred directions (0, 90, 180, 270) and plots all tuning curves together
- Adds noise to show realistic tuning (noisy firing rates)

**demo_population_coding.m** -- Script that:
- Uses rng(42) for reproducibility
- Creates 8 neurons with evenly spaced preferred directions (0:45:315)
- Simulates population response to a stimulus at 60 degrees (cosine tuning + noise)
- Plots population activity as a heatmap using imagesc (neurons x 1, with colorbar)
- Calls decode_direction() to compute the population vector
- Plots the population vector with quiver() alongside the true stimulus direction
- Prints "Actual: X deg, Decoded: Y deg, Error: Z deg"
- Tests multiple stimulus directions and plots decoded vs actual (scatter plot, reference line)

**decode_direction.m** -- Function file:
- function decoded_dir = decode_direction(responses, pref_dirs)
- Implements population vector: weighted sum of preferred directions using cosd/sind
- Returns decoded direction using atan2d
- Includes help comment block explaining the function
